**Releases.** Tagging `v*` triggers the release workflow, which cross-compiles the pure platform CLI binaries (`cc-notes_<os>_<arch>`), builds/signs/notarizes the fixed `CCNotesHelper.app` through the pinned reusable app workflow, verifies the exact helper asset URL and digest, generates `SHA256SUMS`, and uploads the complete release set. The reusable app workflow only emits a verified Actions artifact; after every CLI, helper, and smoke gate passes, the caller creates one draft release, verifies its exact complete asset set, and then publishes it. The tag and helper digest are injected into the CLI through `internal/version`; `cc-notes package install` consumes only the Homebrew formula's verified local helper resource and installs it at `~/Applications/CCNotesHelper.app`, while `cc-notes service install` activates an already-installed helper. Tag merged commits on `main` (e.g. `git tag vX.Y.Z origin/main`), not a feature branch. On stable tags (no `-` suffix) the `bump-formula` job renders the Homebrew formula from the canonical inline template in `release.yml`, fills in the tag version and CLI asset digests, and publishes it through the shared `yasyf/homebrew-tap` action so `brew install yasyf/tap/cc-notes` tracks the latest stable release. Prerelease tags skip the formula. Never re-tag a published stable version: immutable helper and CLI asset digests are part of the deployment contract.
